Orca is an agent platform for teams that need reusable agents, typed APIs, streaming runs, and published chat experiences — all from a single control surface. You define an agent profile once, describing its model, runtime, system prompt, tools, and integrations, and then invoke it from the dashboard, CLI, SDKs, or directly over HTTP. Orca handles the orchestration, streaming, session continuity, and publishing so you can focus on what the agent actually does.

## What you can build

- Product features that submit agent work and stream progress to users.
- Internal automation that creates profiles, launches runs, and reads usage.
- Support, research, operations, coding, and workflow agents with reusable configuration.
- Public chat endpoints for selected published agents.

## Main surfaces

| Surface | Use it for |
| --- | --- |
| Dashboard | Create and manage agents visually |
| CLI | Operate Orca from a terminal or script |
| SDKs | Integrate Orca into product code and services |
| Control plane API | Manage agents, runs, sessions, skills, tools, storage, memory, keys, and workflows |
| Chat gateway API | Send end-user chat messages to published agents |

## Concepts at a glance

**Agents** are reusable profiles: runtime, model, instructions, tools, skills, and optional MCP servers.

**Runs** are units of work submitted to an agent. They stream progress, assistant output, tool activity, usage, and terminal results.

**Sessions** preserve continuity across related runs when you pass the same session ID.

**Published agents** expose a stable chat API without exposing the authoring or control-plane surface.
