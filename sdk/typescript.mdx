The Orca TypeScript SDK gives you a fully typed client for the Orca control-plane API. You can use it to define profiles, submit runs, stream real-time output, and manage sessions, skills, storage, and memory — all from Node.js services or browser applications. The client is built around native `async`/`await` and async iteration, so it fits naturally into modern TypeScript codebases.

## Installation

Install the package with your preferred package manager:

<CodeGroup>

```bash npm
npm install @okikorg/orca
```

```bash yarn
yarn add @okikorg/orca
```

```bash pnpm
pnpm add @okikorg/orca
```

</CodeGroup>

## Client setup

Import `OrcaClient` and instantiate it once — typically at application startup or in a shared module. The client reads your credentials from environment variables:

```typescript
import OrcaClient from '@okikorg/orca';

const client = new OrcaClient({
  baseUrl: process.env.ORCA_API_URL,
  apiKey: process.env.ORCA_API_KEY,
  tenantId: process.env.ORCA_TENANT_ID,
});
```

<Note>
  Set `ORCA_API_URL`, `ORCA_API_KEY`, and `ORCA_TENANT_ID` in your environment before running your application. Do not commit these values to source control.
</Note>

## Create a profile

A profile defines a reusable agent: its runtime, model, system instructions, and available tools. Call `client.profiles.create` to register one:

```typescript
await client.profiles.create({
  name: 'researcher',
  runtime: 'general',
  model: 'anthropic:claude-opus-4-6',
  systemPrompt: 'You are a research assistant. Be concise and cite sources.',
  tools: ['@default'],
});
```

You only need to create a profile once. After that, you can submit runs against it by name.

## Submit a run

Call `client.runs.create` with the profile name and a prompt to start a run. The response includes a `runId` and a `sessionId` you can use to continue the conversation later:

```typescript
const run = await client.runs.create({
  profile: 'researcher',
  title: 'SDK onboarding',
  prompt: 'Explain Orca profiles and runs in three bullets.',
});

console.log('Run ID:', run.runId);
console.log('Session ID:', run.sessionId);
```

## Stream events

After creating a run, stream its events with `client.runs.stream`. The stream is an async iterable — each iteration yields one event object:

```typescript
const stream = await client.runs.stream({ id: run.runId });

for await (const event of stream) {
  if (event.type === 'assistant' || event.type === 'result') {
    if (event.message) {
      process.stdout.write(event.message);
    }
  }
}
```

## Handle events

Use a `switch` statement to handle each event type and drive your application logic accordingly:

```typescript
const stream = await client.runs.stream({ id: run.runId });

for await (const event of stream) {
  switch (event.type) {
    case 'assistant':
      // Incremental assistant output — stream to your UI or buffer it
      if (event.message) process.stdout.write(event.message);
      break;

    case 'tool_call':
      // A tool is being invoked — log or display tool activity
      console.log(`\n[tool] ${event.toolName}`);
      break;

    case 'result':
      // The run has completed — event.message contains the final output
      if (event.message) console.log('\n[result]', event.message);
      break;

    case 'error':
      // The run encountered an error
      console.error('[error]', event.message);
      break;

    default:
      // Ignore other event types (usage, terminal, etc.)
      break;
  }
}
```

<Tip>
  Pass `sessionId: run.sessionId` in your next `client.runs.create` call to continue the same conversation. Orca uses the session ID to preserve context across related runs, enabling multi-turn workflows without re-sending history.
</Tip>
